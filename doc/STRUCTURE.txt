キーワードから探す:
=================
「プロジェクト構成を知りたい」
  → README.md → 00_project_overview.md

「ビルドしたい」
  → README.md → 01_setup_guide.md, 04_build_and_run.md

「テストを実行したい」
  → 04_build_and_run.md

「テストケースを追加したい」
  → 02_module_tests.md → 03_fff_guide.md

「外部モジュールをモック化したい」
  → 03_fff_guide.md

「カバレッジレポートを見たい」
  → 04_build_and_run.md（カバレッジ測定セクション）

「エラーが出た」
  → 01_setup_guide.md（トラブルシューティング）
  → 04_build_and_run.md（一般的な問題と解決方法）

各ドキュメントの特徴:
==================
✅ 目次付き - 各セクションに記号付きヘッダー
✅ テーブル付き - 参照しやすい比較表
✅ コード例付き - すぐに試せるサンプル
✅ リンク付き - 関連ドキュメントへの相互参照
✅ チェックリスト - 実行前の確認リスト
✅ 図説付き - ツリー図表で構成を可視化

使用技術:
========
- GTest: C++ テストフレームワーク
- FFF: Fake Functions Framework（スタブ化）
- CMake: ビルドシステム
- gcov: コードカバレッジ測定
- CTest: テスト実行・管理

プロジェクト構造:
================
target/           → 本体モジュール（テスト対象）
  ├── inc/        → 共通ヘッダー
  └── jchg/       → 各モジュール実装

tests/            → テストファイル
  ├── moduleA/
  ├── moduleB/
  └── moduleC/    ← 実装例

double/           → スタブ（将来の拡張用）
fff/              → FFF フレームワーク
GoogleTest/       → GTest フレームワーク
